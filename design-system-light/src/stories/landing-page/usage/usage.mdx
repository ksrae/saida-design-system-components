### Step 2: Style Sheet Configuration
To handle SCSS files in your project, first install the SASS package:
```bash
npm install sass
```

#### Current Method (will be deprecated)
Import `sy-styles.css` files into your style.scss file.
```scss
@import '@dc/design-system-core/dist/sy-styles.css';
```

If you encounter a warning like the one below, it is recommended to replace the deprecated @import statements with @use

```
▲ [WARNING] Deprecation [plugin angular-sass]

    src/styles.scss:2:8:
      2 │ @import "@dc/design-system-core/dist/style/sy-styles.scss";
        ╵         ^

  Sass @import rules are deprecated and will be removed in Dart Sass 3.0.0.

  More info and automated migrator: https://sass-lang.com/d/import

  The plugin "angular-sass" was triggered by this import

    angular:styles/global:styles:1:8:
      1 │ @import 'src/styles.scss';
        ╵         ~~~~~~~~~~~~~~~~~
```

#### Recommended Solution
Replace the deprecated `@import` statements with `@use`
```scss
@use "@dc/design-system-core/dist/sy-styles.css" as systyle;
```

#### Note 
- The `@use` rule is supported in Sass versions 1.23.0 and later
- `@import` will be deprecated in Dart Sass 3.0.0

### Step 4: Data/Property binding & Event Handling.
Bind properties as you would bind state in React. 
Use curly braces {} to bind variable values to synergy attributes in React.

For complex property assignments, it is recommended to use useEffect.

Unlike Vue and Angular, React uses synthetic events. 
To handle custom events emitted by web components, you need to use `addEventListener` to directly access the DOM elements.

```javascript
import '@dc/design-system-core/dist/sy-components.es.js';
import { useEffect } from 'react'
function App() {

  const autocompleteRefs = useRef(null);
  const buttonRef = useRef(null);

  useEffect(() => {
  const button = buttonRef.current;
    if (button) {
      const handleButtonClick = (event) => {
        console.log('Button clicked:', event);
      };
      button.addEventListener('click', handleButtonClick);

      return () => {
        button.removeEventListener('click', handleButtonClick);
      };
    }}, []);

  useEffect(() => {
    autocompleteRefs.current.source = ["Apple", "Banana", "Cherry", "Date", "Elderberry"];
  }, []);

  return (
    <div className="App">
      <sy-button ref={buttonRef}>Button</sy-button>
      <sy-input-number decimalplaces={0} rounding="round" value={10.5}></sy-input-number>
      <sy-autocomplete ref={autocompleteRef}></sy-autocomplete>
    </div>
  )
}

export default App;
```

### Step 1: HTML Configuration
```html
<!DOCTYPE html>
<html lang="en">
<head>
   <meta charset="UTF-8">
   <title>My Node.js Server</title>
   <link href="node_modules/@dc/design-system-core/dist/sy-styles.css" rel="stylesheet" />
</head>
<body class="sy-theme sy-theme-light sy-theme-default sy-appname">
    <script type="module" src="node_modules/@dc/design-system-core/dist/sy-components.es.js"></script>
</body>
</html>
```

If the test environment is not a web server and testing on web browser only.
Copy the `sy-components.es.js` and `sy-styles.css` files from the `node_modules/@dc/design-system-core/dist/` directory to `public` directory of the project.
and change the script and link tags in the HTML file to point to the copied files:

```html
<link href="./sy-styles.css" rel="stylesheet" />
<script type="module" src="./sy-components.es.js"></script>
```

It is becasuse the browser does not allow to load modules from local file system.

const express = require('express');

  res.sendFile(path.join(__dirname, 'index.html'));

## Cherry Picking

We offer both loading all components at once via `@dc/design-system-core/dist/sy-components.es.js` **and** cherry-picking to import only the components you need.
Cherry-picking components reduces bundle size, improves loading speed, optimizes network usage, eliminates unused code, and lowers memory consumption.

You need to import all required components. For example, `sy-dropdown` includes `sy-menu`, so to use the dropdown component, you should import them as shown below.

```ts
import '@dc/design-system-core/dist/components/dropdown';
import '@dc/design-system-core/dist/components/menu';

```

## Use the token of the Design System
### Step 1: Import `global.scss`
To import `global.scss` file to use the Design System tokens, importing `sy-styles.css` file is required.
Please make sure `sy-styles.css` file is imported in the project.

```html
<link rel="stylesheet" href="@dc/design-system-core/dist/sy-styles.css">
```

### Step 2: Import `global.scss` file
```scss
@import '@dc/design-system-core/dist/style/global.scss';
```

or use @use
```scss
@use '@dc/design-system-core/dist/style/global.scss' as globalstyle;
```

### Step 3: Use the token in your styles
```scss
@use '@dc/design-system-core/dist/style/global.scss' as globalstyle;
.my-component {
  color: globalstyle.$color-primary;
  background-color: globalstyle.$background-secondary;
}
```
