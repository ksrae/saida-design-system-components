## Compatibility

### Tab

There is a significant change to the tab component of the Version 3.0.0.<br/>
This change was for improving functionality for easier use of tabs and to enhance extensibility for various tab applications.<br/>
Please note that versions after 3.0.0 are not compatible with the structure of previous 2.x.x versions.<br/>
- Each sy-tab no longer applies slot=title. Instead, you should set slot=tabs on the parent container that holds the sy-tabs.
- To use sy-tab-content, it is requried to define a parent container and set "slot=contents"

**ASIS**
```
<sy-tab-group>
    <sy-tab slot="title" key="t1" ?closable=${args.closable}>tab1</sy-tab>
    <sy-tab slot="title" key="t2" ?closable=${args.closable}>tab2</sy-tab>

    <sy-tab-content name="t1">This is tab1</sy-tab-content>
    <sy-tab-content name="t2">This is tab2</sy-tab-content>
</sy-tab-group>
```

**TOBE**
```
<sy-tab-group>
  <div slot="tabs">
    <sy-tab key="t1" ?closable=${args.closable}>tab1</sy-tab>
    <sy-tab key="t2" ?closable=${args.closable}>tab2</sy-tab>
  </div>
  <div slot="contents">
    <sy-tab-content name="t1">This is tab1</sy-tab-content>
    <sy-tab-content name="t2">This is tab2</sy-tab-content>
  </div>
</sy-tab-group>
```

**Note**: type="navigation" in tab group is deprecated. (It will be delivered as a global header in MVP4)
