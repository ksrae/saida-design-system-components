# v2.0.3
## Updates
### Input
- changed event is fired when clear icon is clicked.


### Tree
- manualRemove Attribute has been added. 
- removeNode function has been added.<br/>
If manualRemove is set, when itemRemoved event is fired, the event returns node value but it does not remove the node.<br/>
A developer can add modal or any notification at this moment.<br/>
If user select to remove, remove the node manually, using removeNode function. 


### Inline message 
- fix bug when using focusout trigger in the inline message.


# v2.0.2
## Updates
### Tab
- Max- width of `<sy-tab>` has been removed which means length of the tab can be expanded as much as length of its slot.<br/>
- Developers can also control its size with adding style. But changing any other style may affacts its feature or style.

```
<sy-tab style="width:100px;">tab 1</sy-tab>
```

### Tree
- "itemRemoved" event will be fired, when a tree node is removed by user clicking remove-icon.


### Tooltip
- itemSelected event files once only at clicking an item. 
- linefeed is necessary Fixed
- content allows special characters such as \n for newline, \t for tab.
- HTML are not allowed. It will be shown as plain text.   
- text is not wrapped Fixed
- content will be wrapped by tooltip size.
- Default width of the tooltip is 200px.

- Enhance style: Distinguish enable/disable clearly

- Resolve the issue of the text line in the button.

# v2.0.1
## Updates
### Step
- Fixed an issue where only the number of steps specified by 'step' stayed active when a startindex was set, preventing any steps beyond that from activating. <br/>
Modified so the defined 'step' count remains active even with a startindex set.

- sy-steps and sy-step must be composed as 1-depth and should not be placed in a 2-depth hierarchy.<br/>
However, if composed within Angulars invisible DOM (e.g., `<ng-container>`), it was actually 1-depth but not displayed correctly. 

### Dropdown
- does not show on the modal fixed<br/>
zIndex has been reorganized.

### Tab
- tab scroll overflow error

### Font
- Limit the scope of root font style to Synergy components only
