# Variant 
## Definition
There are four types of inline messages: `info`, `success`, `error`, and `warning`.

## Values
### Info (default)
Offers helpful guidance or additional context related to a specific UI element.
Often used in forms or tables to assist users in completing a task.<br/>
This variant is noticeable enough to provide helpful context but not disruptive. <br/>


### Success
Confirms that the user’s input or action was successful within a particular component or form.	<br/>


### Error
Informs the user that there is a problem with their input or an action they tried to complete. <br/>
The user must resolve the issue before proceeding.<br/>


### Warning	
Warns the user about potential issues with the current state of their input or interaction, suggesting that something might need attention soon.<br/>
