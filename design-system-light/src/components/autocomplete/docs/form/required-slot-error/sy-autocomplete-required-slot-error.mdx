# Required + Slot-based custom error

The canonical pattern for inline error messages across all SAIDA form-associated components:

```html
<sy-autocomplete required trigger="input">
  <div slot="error">
    <p>🚫 Custom error: You must select a valid fruit!</p>
  </div>
</sy-autocomplete>
```

Behaviour:

1. Declarative slot — author writes the error UI once.
2. When any native constraint fails (here, `required`) **and** the user has touched the
   field or submitted the form, the slot content appears below the field.
3. The native browser validation bubble is suppressed so the inline slot is the only
   error surface.
4. `el.reportValidity()` returns `false`; `el.validationMessage` equals the slot's text
   so `FormData` consumers and server-side error logs stay informative.
