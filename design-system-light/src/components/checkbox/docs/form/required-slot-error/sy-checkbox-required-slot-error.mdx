# Required + Slot-based custom error

The canonical pattern for inline error messages on `sy-checkbox`, identical to
`sy-autocomplete` and `sy-input`:

```html
<sy-checkbox required>
  I accept the terms and conditions
  <div slot="error">
    <p>🚫 You must accept the terms to continue.</p>
  </div>
</sy-checkbox>
```

Behaviour:

1. Declarative slot — author writes the error UI once.
2. When `required` fails **and** the user has touched the field or submitted
   the form, the slot content appears below the checkbox.
3. The native browser validation bubble is suppressed so the inline slot is the only
   error surface.
4. `el.reportValidity()` returns `false`; `el.validationMessage` equals the slot's text
   so `FormData` consumers and server-side logs stay informative.
