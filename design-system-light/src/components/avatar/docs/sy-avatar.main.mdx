# Avatar

## Definition
`sy-avatar` visually represents a person or object. By default, it shows ≤2 initial characters
from the user's name. You can personalise it with icons, manual initials, or profile images.

Content rendering priority (highest → lowest): **image → icon → text → letter**.

## When to use
- Identify users consistently across conversations, listings, comments, notifications.
- Personalise the UI with a visible sense of identity.

## When not to use
- No personalisation is involved — the avatar becomes visual noise.
- No meaningful representation is available (empty placeholders do not help users).

## Accessibility
- Non-clickable avatars render with `role="img"` and an `aria-label` derived from
  `tooltipContent` (or, when unset, from image/icon/text/letter in that priority).
- Clickable avatars render with `role="button"`, are keyboard-activable (Enter / Space),
  and show a focus outline.
- Disabled avatars carry `aria-disabled="true"` and don't receive keyboard focus.

# Avatar Group
## Definition
Avatar Group renders its children as a stack by default. Use `maxCount` to cap the number of
visible avatars. (See the separate `avatar-group` spec page — that component is scheduled for
