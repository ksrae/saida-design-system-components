# Required + Slot-based custom error

The canonical pattern for inline error messages on `sy-datepicker`, identical to
`sy-autocomplete`, `sy-checkbox`, and `sy-input`:

```html
<sy-datepicker required name="deliveryDate" placeholder="Pick a delivery date">
  <div slot="error">
    <p>🚫 Please select a delivery date.</p>
  </div>
</sy-datepicker>
```

Behaviour:

1. Declarative slot — author writes the error UI once.
2. When `required` fails **and** the user has touched the field or submitted
   the form, the slot content appears below the datepicker.
3. The native browser validation bubble is suppressed so the inline slot is the only
   error surface.
4. `el.reportValidity()` returns `false`; `el.validationMessage` equals the slot's text.
