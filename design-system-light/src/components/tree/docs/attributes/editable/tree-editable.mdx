# Editable
## Definition
All tree items are set to be editable. 
 three options provides when clicking a editable node:
- edit: Edits the label of the node. It emits the itemEdited event.
- add (+): Adds a child to the node. The value is added as "(parent's value)-(current time)" to ensure uniqueness. It emits the itemAdded event.
- remove (-): Removes the node along with all its child nodes. It emits the itemRemoved event.

By modifying the editable, appendable, and removable values of each node, you can apply or not apply all or some of the editable features.


```json
{
	label: 'item1',
	value: '1', 
	editable: false, 		// set true to show edit button. default is true
	appendable: false,	// set true to show add button. default is true
	removable: false		// set true to show remove button. default is true
}
```

## Boolean Attribute Handling
In the source code, <b>editable = ""</b> should be modified as follows
- Set `editable` attribute and don't put a value after it.
```
<sy-tree editable></sy-tree>
```
- Alternatively, you can explicitly set it to true, but to not apply `editable`, remove the attribute instead of setting its value to false.

```
<sy-tree editable="true"></sy-tree>
```
